# Руководство администратора

Это руководство описывает основные операции для администратора платформы Апекс.

## Вход в админ-панель

1. Откройте `http://ваш-домен/admin/`
2. Войдите с учётными данными администратора
3. По умолчанию после сидинга: `admin@apeks.space` / `admin123`

## Создание курса

1. В админ-панели перейдите в раздел **Courses → Courses**
2. Нажмите **Add Course**
3. Заполните:
   - **Title** — название курса (например, «Всерос с Апексом 25/26»)
   - **Slug** — URL-идентификатор (автоматически генерируется из названия)
   - **Description** — описание (поддерживает Markdown)
   - **Is published** — поставьте галочку, чтобы курс был виден ученикам
4. Сохраните

## Создание структуры курса

### Блоки

1. В карточке курса внизу есть секция **Blocks** (inline)
2. Добавьте блоки, указав название и порядковый номер (`order`)
3. Или перейдите в **Courses → Blocks** и создайте блок, выбрав курс

### Темы

1. Перейдите в **Courses → Topics** или в карточку блока
2. Создайте тему, привязав к блоку
3. Укажите название и порядковый номер

### Занятия

1. Перейдите в **Courses → Lessons** или в карточку темы
2. Создайте занятие:
   - **Title** — название
   - **Topic** — выберите тему
   - **Lesson date** — дата проведения
   - **Video provider** — оставьте YouTube
   - **Video url** — вставьте ссылку на YouTube-видео (полный URL)
   - **Description** — описание (Markdown, LaTeX через `$...$`)
   - **Notes file** — загрузите PDF-конспект
   - **Workbook file** — загрузите PDF рабочей тетради
   - **Is published** — галочка для публикации
   - **Order** — порядковый номер внутри темы

## Создание домашнего задания

1. Перейдите в **Homework → Homeworks**
2. Нажмите **Add Homework**
3. Выберите **Lesson** (занятие, к которому привязывается ДЗ)
4. Укажите **Title** (опционально)
5. Сохраните

### Добавление задач

1. Откройте созданное ДЗ
2. В секции **Problems** (inline) добавьте задачи
3. Или перейдите в **Homework → Problems** → **Add Problem**
4. Для каждой задачи укажите:
   - **Order** — порядковый номер
   - **Statement** — условие (Markdown + LaTeX: `$формула$`)
   - **Answer type** — тип ответа:
     - `text` — текстовый (ручная проверка)
     - `choice_single` — один вариант
     - `choice_multiple` — несколько вариантов
     - `number` — числовой
     - `formula` — формула
   - **Correct answer** — JSON с правильным ответом (см. ниже)
   - **Max score** — максимальный балл
   - **Hint** — подсказка (опционально)

### Формат correct_answer

**Для choice_single:**
```json
{"correct_option_id": "ID_варианта"}
```
(ID заполняется после создания вариантов)

**Для choice_multiple:**
```json
{"correct_option_ids": ["ID1", "ID2"]}
```

**Для number:**
```json
{"value": 42.5, "tolerance_type": "abs", "tolerance": 0.5}
```
- `tolerance_type`: `"abs"` (абсолютная) или `"rel"` (относительная)

**Для formula:**
```json
{"latex": "\\frac{4\\pi^2 a^3}{G(M+m)}"}
```

### Добавление вариантов ответа (для choice)

1. Откройте задачу
2. В секции **Problem options** добавьте варианты
3. Отметьте правильные галочкой **Is correct**

### Добавление картинок к задаче

1. Откройте задачу
2. В секции **Problem attachments** загрузите файлы

## Выдача доступа ученику

### Через карточку пользователя

1. Перейдите в **Core → Users**
2. Найдите пользователя (поиск по email/имени)
3. Откройте карточку
4. В секции **Enrollments** (inline) добавьте запись:
   - Выберите курс
   - Сохраните

### Через API

```
POST /api/admin/users/{user_id}/enroll
Body: {"course_id": 1}
```

## Отзыв доступа

1. В карточке пользователя удалите запись из секции Enrollments
2. Или через API: `DELETE /api/admin/users/{user_id}/enroll/{course_id}`

## Ручная установка прогресса

1. Перейдите в **Courses → Enrollments**
2. Найдите нужную запись (ученик + курс)
3. Установите **Manual progress override** (0-100)
4. Это значение будет показано ученику вместо автоматически рассчитанного

## Проверка домашних заданий

1. Перейдите в **Homework → Submissions**
2. Отфильтруйте по `is_auto_checked = False` (ручная проверка)
3. Откройте сдачу
4. Посмотрите ответ ученика в поле **Answer**
5. Заполните:
   - **Score** — балл (от 0 до max_score задачи)
   - **Is correct** — правильно/неправильно
   - **Admin comment** — комментарий для ученика
   - **Checked at** — дата проверки (заполнится автоматически через API)
6. Сохраните

## Управление календарём олимпиад

1. Перейдите в **Calendar_app → Olympiad events**
2. Создайте событие:
   - **Title** — название
   - **Event type** — тип (выберите из списка или создайте новый)
   - **Start date** — дата начала
   - **End date** — дата окончания (для многодневных)
   - **Time** — время (опционально)
   - **Description** — описание (Markdown)
   - **External url** — ссылка на сайт олимпиады

### Управление типами событий

1. **Calendar_app → Olympiad event types**
2. Создайте тип: название + цвет (hex, например `#3B82F6`)

## Блокировка пользователя

1. Откройте карточку пользователя
2. Снимите галочку **Is active**
3. Пользователь не сможет войти

## Назначение администратора

1. Откройте карточку пользователя
2. Поставьте галочки **Is admin** и **Is staff**
3. Сохраните

## Полезные команды

```bash
# Создать суперпользователя
docker-compose exec backend python manage.py createsuperuser

# Загрузить тестовые данные
docker-compose exec backend python manage.py seed_data

# Бэкап базы данных
docker-compose exec db pg_dump -U apeks apeks > backup_$(date +%Y%m%d).sql

# Восстановление из бэкапа
docker-compose exec -T db psql -U apeks apeks < backup_20250101.sql
```
